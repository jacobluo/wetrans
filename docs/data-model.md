# wetrans Data Model

Status: Draft for review
Source PRD: `docs/prd.md`

## 1. Purpose

This document defines the MVP data model for wetrans.

It separates:

- Persisted app data.
- Keychain secrets.
- Runtime-only session state.
- Transfer task state.

The goal is a data model that is stable enough for implementation, testable without a live server, and easy to migrate.

## 2. Storage Overview

```text
~/Library/Application Support/wetrans/
  hosts.json
  known_hosts.json
  transfer_history.json
```

Keychain:

```text
service: wetrans.ssh.password
account: <hostId>

service: wetrans.ssh.keyPassphrase
account: <hostId>
```

Runtime memory only:

- Live SSH/SFTP sessions.
- Current loading states.
- AppKit selection objects.
- Directory scroll positions.
- Expanded directory nodes.

## 3. ID Strategy

Use `UUID` for saved host IDs and transfer task IDs.

Rules:

- `SavedHost.id` is generated by wetrans.
- SSH Config aliases are not primary keys.
- Manual host names are not primary keys.
- Transfer tasks reference `hostId`.
- Keychain items use `hostId.uuidString` as account.

## 4. hosts.json

### 4.1 Document Shape

```json
{
  "schemaVersion": 1,
  "hosts": []
}
```

### 4.2 SavedHost JSON

```json
{
  "id": "8C765C99-6B46-4D74-BB3C-2B71F21997C6",
  "source": "sshConfigGenerated",
  "displayName": "dev",
  "hostname": "192.0.2.10",
  "port": 22,
  "username": "ubuntu",
  "authType": "sshKey",
  "identityFile": "/Users/example/.ssh/id_ed25519",
  "isFavorite": true,
  "lastConnectedAt": "2026-06-26T08:00:00Z",
  "lastRemotePath": "/home/ubuntu/project",
  "lastLocalPath": "/Users/example/Downloads",
  "defaultRemotePath": "/home/ubuntu",
  "favoriteRemotePaths": [
    "/home/ubuntu/project",
    "/var/log"
  ],
  "originSSHConfigAlias": "dev",
  "resolvedAt": "2026-06-26T07:50:00Z",
  "note": "Development server"
}
```

### 4.3 SavedHost Swift Model

```swift
enum HostSource: String, Codable {
    case manual
    case sshConfigGenerated
}

enum AuthType: String, Codable {
    case password
    case sshKey
}

struct SavedHost: Identifiable, Codable, Equatable {
    let id: UUID
    var source: HostSource

    var displayName: String
    var hostname: String
    var port: Int
    var username: String
    var authType: AuthType
    var identityFile: String?

    var isFavorite: Bool
    var lastConnectedAt: Date?
    var lastRemotePath: String?
    var lastLocalPath: String?
    var defaultRemotePath: String?
    var favoriteRemotePaths: [String]

    var originSSHConfigAlias: String?
    var resolvedAt: Date?
    var note: String?
}
```

### 4.4 Validation Rules

- `displayName` must be non-empty.
- `hostname` must be non-empty.
- `port` must be between 1 and 65535.
- `username` must be non-empty.
- `identityFile` is required when `authType == .sshKey` unless later design adds default key discovery.
- `originSSHConfigAlias` is optional metadata and must not be required for connection.
- `favoriteRemotePaths` should not contain duplicates.

### 4.5 Sensitive Data Rule

`hosts.json` must not contain:

- SSH password.
- Private key passphrase.
- Token-like secrets.

## 5. Keychain Records

### 5.1 Password

```text
service: wetrans.ssh.password
account: <hostId.uuidString>
value: UTF-8 password bytes
```

### 5.2 Private Key Passphrase

```text
service: wetrans.ssh.keyPassphrase
account: <hostId.uuidString>
value: UTF-8 passphrase bytes
```

### 5.3 Lifecycle Rules

- Create or update credentials when saving host auth settings.
- Delete related credentials when deleting a host.
- If auth type changes from password to SSH key, delete password unless the user chooses to keep it.
- If auth type changes from SSH key to password, delete key passphrase unless the user chooses to keep it.

## 6. known_hosts.json

### 6.1 Document Shape

```json
{
  "schemaVersion": 1,
  "trustedHostKeys": []
}
```

### 6.2 TrustedHostKey JSON

```json
{
  "id": "B57834C5-251C-4AF4-81C4-7E7407B3D02A",
  "hostId": "8C765C99-6B46-4D74-BB3C-2B71F21997C6",
  "hostname": "192.0.2.10",
  "port": 22,
  "keyType": "ssh-ed25519",
  "fingerprintSHA256": "SHA256:example",
  "firstTrustedAt": "2026-06-26T08:00:00Z",
  "lastVerifiedAt": "2026-06-26T08:00:00Z"
}
```

### 6.3 TrustedHostKey Swift Model

```swift
struct TrustedHostKey: Identifiable, Codable, Equatable {
    let id: UUID
    let hostId: UUID
    var hostname: String
    var port: Int
    var keyType: String
    var fingerprintSHA256: String
    var firstTrustedAt: Date
    var lastVerifiedAt: Date
}
```

### 6.4 Trust Rules

- Unknown key: ask user to trust or cancel.
- Matching trusted key: continue and update `lastVerifiedAt`.
- Changed key: block connection and show strong warning.
- Trust records are scoped to wetrans and do not modify OpenSSH `known_hosts`.

## 7. transfer_history.json

Each selected file becomes one `TransferTask`. A multi-file upload or download is represented as multiple task records in the same global queue rather than a nested batch object in MVP.

### 7.1 Document Shape

```json
{
  "schemaVersion": 1,
  "tasks": []
}
```

### 7.2 TransferTask JSON

```json
{
  "id": "AFB0EE91-3E6D-4B43-B3E5-00F9551162A6",
  "hostId": "8C765C99-6B46-4D74-BB3C-2B71F21997C6",
  "hostDisplayName": "dev",
  "direction": "upload",
  "localPath": "/Users/example/Downloads/config.yaml",
  "remotePath": "/home/ubuntu/project/config.yaml",
  "fileName": "config.yaml",
  "totalBytes": 8421,
  "transferredBytes": 8421,
  "progress": 1.0,
  "speedBytesPerSecond": 1200000,
  "status": "succeeded",
  "errorMessage": null,
  "createdAt": "2026-06-26T08:05:00Z",
  "startedAt": "2026-06-26T08:05:01Z",
  "completedAt": "2026-06-26T08:05:03Z"
}
```

### 7.3 Transfer Swift Model

```swift
enum TransferDirection: String, Codable {
    case upload
    case download
}

enum TransferStatus: String, Codable {
    case pending
    case running
    case succeeded
    case failed
    case cancelled
    case paused
}

struct TransferTask: Identifiable, Codable, Equatable {
    let id: UUID
    let hostId: UUID
    let hostDisplayName: String

    let direction: TransferDirection
    let localPath: String
    let remotePath: String

    let fileName: String
    let totalBytes: UInt64?
    var transferredBytes: UInt64
    var progress: Double

    var speedBytesPerSecond: UInt64?
    var status: TransferStatus
    var errorMessage: String?

    let createdAt: Date
    var startedAt: Date?
    var completedAt: Date?
}
```

### 7.4 Persistence Rules

- Persist completed, failed, and cancelled summaries.
- Persist pending tasks only if they are not tied to a live process.
- On app startup, any persisted `running` task should be converted to `failed` with an interrupted message.
- Keep a bounded history in MVP, for example the latest 500 tasks.

## 8. Runtime HostSessionState

Runtime state should not be persisted in full during MVP.

```swift
struct HostSessionState {
    let hostId: UUID

    var isConnected: Bool
    var lastActiveAt: Date?

    var currentRemotePath: String
    var currentLocalPath: String

    var expandedRemotePaths: Set<String>
    var selectedRemotePaths: Set<String>
    var selectedLocalPaths: Set<String>

    var remoteScrollPosition: Double?
    var localScrollPosition: Double?
}
```

Persisted subset:

- `currentRemotePath` -> `SavedHost.lastRemotePath`
- `currentLocalPath` -> `SavedHost.lastLocalPath`

Runtime-only subset:

- `isConnected`
- expanded paths
- selected paths
- scroll positions
- loading state
- error presentation state

## 9. FileItem

Used by both local and remote file panels.

```swift
struct FileItem: Identifiable, Equatable {
    let id: String
    let name: String
    let path: String
    let isDirectory: Bool
    let isSymlink: Bool
    let size: UInt64?
    let modifiedAt: Date?
    let permissions: String?
}
```

Rules:

- `id` should be stable within a directory listing.
- For MVP, use normalized absolute path as `id`.
- Remote permissions can be nil if the SFTP adapter cannot read them.

## 10. HostDraft

`HostDraft` is a UI/editing model and should not be persisted directly.

```swift
struct HostDraft {
    var source: HostSource
    var displayName: String
    var hostname: String
    var port: Int
    var username: String
    var authType: AuthType
    var identityFile: String?
    var password: String?
    var keyPassphrase: String?
    var defaultRemotePath: String?
    var originSSHConfigAlias: String?
    var resolvedAt: Date?
    var unsupportedOptions: [SSHConfigWarning]
}
```

Rules:

- Convert `HostDraft` to `SavedHost` only after validation.
- Save `password` and `keyPassphrase` to Keychain, never to `SavedHost`.
- Preserve unsupported SSH options only as warning text unless later needed.

## 11. Migration Strategy

Every persisted JSON document has `schemaVersion`.

MVP starts at:

```text
schemaVersion: 1
```

Migration rules:

- Unknown future schema versions should fail with a readable error.
- Missing optional fields should use defaults.
- Migrations should be pure functions with unit tests.
- Before writing migrated data, keep a backup copy during development builds.

## 12. Atomic Write Strategy

For JSON files:

1. Encode to memory.
2. Write to a temporary file in the same directory.
3. Flush to disk.
4. Rename over the target file.

This avoids partial writes if the app quits mid-save.

## 13. Privacy and Security Notes

- Do not log secrets.
- Do not include passwords or passphrases in debug details.
- Do not include full private key contents anywhere.
- Treat local paths and remote paths as user data.
- Avoid sending any telemetry in MVP unless separately designed and approved.
